OSLC-RM demo services...

This is the first version of some REST services implementing the specification of OSLC-RM.

Vocabulary Management (on-going work)

In order to implement the OSL specification with regards to Requirements Management we should be able to share the vocabulary.

That is why we have select a test vocabulary "The Common Procurement Vocabulary" to implement some REST services.

You can access/test these services in the following URIs:

  • Catalog of Services
  • Service Providers and Capabilities
  • Get a vocabulary element (37322000)
  • Get the full vocabulary (not recommend more than 10K concepts)
  • Get a requirement (1)
  • Get the full list of requirements
  • Query Capabilities (on-going)
  • SPARQL interface (TO-DO)
  • Auth (TO-DO)
